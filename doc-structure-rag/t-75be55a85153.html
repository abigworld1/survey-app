どんなもの？

従来のRAGシステムは、表を単一の単位として扱うか、平坦なテキストに変換する粗粒度な戦略を採用しているため、セルレベルの精密な関係性や、表と周囲のテキストとの整合性を捉えきれず、複雑な推論や異種ドキュメント間の情報融合に限界がある。本研究は、表のセマンティクス理解が不十分であるという課題に対し、表をエントリレベルのセマンティック単位に分解して構造化グラフを構築するアプローチをとる。具体的には、テキストと表が混在する複雑な文書において、表のセル単位（entry-level）に焦点を当てた「Fine-grained Table-aware RAG」タスクを定義している。このタスクでは、質問 $Q_i = \langle q_i, f_i \rangle$ に対し、表のエントリ $e^* = (v, c, e, t, s, k, m)$ と、必要に応じてテキスト $p^*$ を検索し、最終的な回答 $A = \text{Inference}(Q, e^*, p^*)$ を生成することを目指す。

Module

FT-RAGは、複雑な表形式データの推論において、構造化されたSAT Graph（Subject, Attribute, Temporal）のナビゲーションと非構造化テキストの融合を行う2段階の検索パラダイムを提案している。第1フェーズの「Hierarchical Graph Navigation」では、クエリの制約に応じて、Subject $s$ と Temporal $t$ から属性を特定する「Anchor-Driven Forward Traversal」と、Attribute $a$ から親ノードを遡る「Attribute-Driven Reverse Traversal」を同期させ、最適な $(s, t, a)$ サブグラフを特定する。さらに、特定された事実タプル $\langle s_i, t_j, a_k \rangle$ に対して、LLMを用いた「Structural Neighbor Expansion」を適用し、時間軸の隣接ノード $(t_{j-1}, t_{j+1})$ 等を拡張することで、比較推論に必要な文脈を確保する。第2フェーズの「Text-Bridged Augmentation」では、取得した構造化回答の集合 $A = \{A_i \mid A_i = (s_i, t_i, a_i, v_i, o_i)\}$ を、テンプレート $T_i$ を用いて自然言語化し、それをクエリとして元のテキストコーパスから関連文脈を埋め込みベースで検索・結合する。実験結果では、既存のStandard RAGやSelf-RAG、MixRAG等の手法と比較して、FT-RAGはHit Rate (HR) において、従来のMixRAGの71.6%を大幅に上回る93.7%を達成しており、高い精度を示している。
