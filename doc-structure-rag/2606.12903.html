Retrieval-Augmented GenerationRAG
一言で: 多言語RAGにおける中国語と英語の証拠衝突を診断するためのベンチマーク $X\text{-RAMDocs-ZHEN}$ と、解釈可能なパイプライン $X\text{-MADAM-RAG}$ を提案する。実験の結果、提案手法は制御された条件下で高い精度を達成するが、回答テンプレートを排除した自然言語ストレス・テストでは性能が大幅に低下し、文書レベルの抽出プロセスが主要なボトルネックであることが明らかになった。

どんなもの？

本研究は、多言語RAGにおいて中国語と英語の検索結果が互いに矛盾する「証拠の衝突（evidence conflict）」が発生した際の、モデルの振る舞いと対処法を調査している。既存のFaithEvalやRAGCheckerといった意味論的な忠実度評価とは異なり、中英間の証拠が「一致（agree）」「不一致（disagree）」「参照言語の逆転（reverse）」のいずれかになるよう制御された設定を用いる。評価対象として、RAMDocsから派生した300の事例と534の証拠ドキュメント（中英各50%）からなる $X\text{-RAMDocs-ZHEN}$ ベンチマークを構築した。このベンチマークは、エンティティ、日付、数値の不一致を網羅しており、情報漏洩を防ぐために正解や条件ラベルを隠蔽した設計となっている。

先行研究と比べてどこがすごい？

第一の貢献は、中英間の証拠衝突を精密に診断するために設計された、制御されたリーク防止型ベンチマーク $X\text{-RAMDocs-ZHEN}$ の提案である。第二に、文書ごとの抽出、証拠の修復、決定論的なグルーピング、衝突を考慮した集約からなる解釈可能なパイプライン $X\text{-MADAM-RAG}$ を提案した。実験を通じて、既存のシングルコール・ベースラインを上回る性能を示す一方で、回答の定型句に依存する脆弱性を特定した。これにより、多言語RAGにおけるハルシネーションの主要な原因が、集約段階ではなく文書レベルの抽出段階にあることを示した。

技術や手法のキモはどこ？

提案手法 $X\text{-MADAM-RAG}$ は、以下の4つの主要ステップからなるパイプラインである。まず、抽出エージェントが各証拠文書から独立して候補を抽出する。次に、抽出に失敗した場合にパース済みフィールド等から候補を補完する「ラベルに依存しない修復（label-blind repair）」を行う。その後、正規化された候補を決定論的にグルーピングし、単一グループなら回答、複数なら衝突の列挙、ゼロなら棄権を出力する。最後に、衝突を認識した集約を行う。評価には、特権的なメタデータを使用する Oracle diagnostic、修復を除去した no-repair、決定論的グルーピングをモデルベースに置き換えた no-grouping のアブレーション解析が用いられる。

どうやって有効だと検証した？

$Qwen2.5\text{-}7B\text{-}Instruct$ を用い、NVIDIA RTX 4090 上で $\text{temperature} = 0$ の決定論的デコーディングを用いて実験を行った。元のベンチマークにおいて、$X\text{-MADAM-RAG}$ は strict accuracy $0.9667$、conflict-aware success $0.9767$ を達成し、evidence-normalized RAGと比較して strict accuracy を $0.0767$ 向上させた。また、unsupported-candidate rate を $0.1067$ から $0.0200$ へと大幅に低下させた。一方で、回答テンプレートを排除した Naturalized Stress Test では、rule-only 抽出器の精度が $1.0000$ から $0.0000$ へ低下する中、$X\text{-MADAM-RAG}$ も $0.3000$ まで低下し、素朴なベースラインを下回る結果となった。

議論はある？（限界・課題）

$X\text{-MADAM-RAG}$ は、repair や決定論的な grouping が性能向上に寄与することを示したが、本手法は汎用的なハルシネーション検出器ではなく、制御された衝突の診断ツールとして位置付けられる。ストレス・テストにおける性能低下と、Oracle が $1.0000$ を維持した事実から、ボトルネックは集約ではなく文書レベルの抽出（extraction）にあることが特定された。本研究の限界として、データが RAMDocs 由来の制御されたスニペットであること、単一モデルへの依存、および自動的な語彙指標による評価であることが挙げられる。今後の課題は、自然な検索環境への適用と、より堅牢な文書レベルの抽出手法の開発である。
