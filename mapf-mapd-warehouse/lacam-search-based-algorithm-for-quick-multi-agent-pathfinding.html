LaCAM: Search-Based Algorithm for Quick Multi-Agent Pathfinding

Keisuke Okumura
arxiv ・ 2022-11-24 ・ source: arxiv
原典
一言で: LaCAM (Lazy Constraints Addition Search for MAPF) は、完全性を備えつつ、数百から10,000規模のエージェントが存在する大規模なマルチエージェント経路探索（MAPF）問題を高速に解くための2層構造の探索アルゴリズムである。

どんなもの？

マルチエージェント経路探索（MAPF）は、グラフ $G = (V, E)$ 上の $n$ 個のエージェント $A = \{1, \dots, n\}$ に対し、各エージェントが開始地点 $s_i$ から目標地点 $g_i$ へ到達する、衝突のない経路集合を求める問題である。本研究が対象とするのは、vertex conflict ($\pi_i[t] = \pi_j[t]$) および swap conflict ($\pi_i[t] = \pi_j[t+1] \land \pi_i[t+1] = \pi_j[t]$) を回避し、解の質として sum-of-costs (SOC) $\sum_{i \in A} T_i$ を最小化することを目指す。既存のサブオプティマルな手法では、高密度な環境や大規模なエージェント数において、30秒のタイムアウト内に解を導出できない課題があった。

先行研究と比べてどこがすごい？

本論文の主な貢献は、高レベルの構成（configuration）探索と低レベルの制約探索を組み合わせ、制約を遅延的に適用することで探索効率を劇的に向上させた LaCAM を提案した点にある。LaCAM は、理論的な完全性を維持しながら、10,000規模のエージェントに対しても30秒以内で解を導出できる極めて高いスケーラビリティを実現した。また、既存の最先端な非最適アルゴリズム（PP, OD, EECBS, LNS2 等）と比較して、成功率、計画時間、および解の品質の観点で同等以上の性能を示すことを実証した。さらに、構成生成に PIBT などの既存手法を組み込むことで、積極的な協調（aggressive coordination）を可能にする新しい探索フレームワークを提示した。

技術や手法のキモはどこ？

LaCAM は、高レベル探索と低レベル探索の2層構造を採用している。高レベル探索は、スタックを用いた深さ優先探索（DFS）スタイルで全エージェントの位置のタプルである configuration の列を探索し、各ノードは制約木（constraint tree）を保持する。低レベル探索は、幅優先探索（BFS）を用いて、エージェント $i$ が現在の頂点 $v$ から隣接頂点 $u \in \text{neigh}(v) \cup \{v\}$ へ移動するという「正の制約」を順次生成する。低レベルの探索深度が $|A|$ に達した際、生成された制約を満たす新しい構成を `get new config` 関数（PIBT 等）を用いて決定し、高レベルの `Open` リストへ追加する。この際、高レベルの探索空間 $O(|V|^{|A|})$ に対して、制約を遅延的に適用することで実質的な分岐係数を抑制している。

どうやって有効だと検証した？

評価実験では、PP, OD, PIBT, PIBT+, EECBS, LNS2 の6つのアルゴリズムを比較対象とした。32 $\times$ 32 グリッド、障害物密度20%、400エージェントの `random-32-32-20` ベンチマークにおいて、既存手法がタイムアウトする中で、LaCAM は中央値1秒で全てのインスタンスを解決した。10,000エージェント規模の `warehouse-20-40-10-2-2` インスタンスにおいても、全てのケースを30秒以内に解決し、高いスケーラビリティを実証した。解の質については、PIBT+ と比較して高密度な状況下で優れた SOC を記録した。一方で、狭い通路でのすれ違いが必要な敵対的インスタンスでは、DFS 形式の探索が類似した構成に捕まり、エージェント数の増加に伴い探索回数が劇的に増大するという限界も確認された。

議論はある？（限界・課題）

LaCAM の高速性の鍵は、高レベルノードにおいて初期状態では後続ノードを1つのみ生成し、必要に応じて制約を追加する「遅延評価（lazy manner）」により、膨大な分岐係数を抑制している点にある。また、構成生成器として PIBT を用いることで、従来の $A^*$ 変種よりも積極的な協調を実現できている。今後の課題として、幅優先探索の導入による makespan の最適化や、大規模インスタンスに対する漸近的最適性（asymptotic optimality）を持つバージョンの開発が挙げられる。さらに、低レベル探索におけるより効果的な制約追加手法や、構成生成器の設計改善が、敵対的インスタンスにおける探索の停滞を克服するために重要である。

Introduction

本論文では、大規模かつリアルタイム性が求められるマルチエージェント経路計画（MAPF）問題に対し、探索効率を劇的に向上させた「Lazy Constraints Addition Search (LaCAM)」を提案している。LaCAMは、全エージェントの配置のタプルである「configuration」を探索するハイレベルな探索と、次回のconfigurationにおけるエージェントの移動制約を探索するローレベルな探索からなる2層構造のアルゴリズムであり、ハイレベルの探索においてローレベルの制約を遅延的に（lazy manner）適用することで探索コストを削減している。理論的には完全性（completeness）を備えており、解が存在する場合は必ず解を返し、存在しない場合はその旨を報告する。実験では、1491 $\times$ 656の広大なグリッドや10,000規模の大量のエージェント、高密度な環境を含む多様なインスタンスにおいて、極めて短時間での解法を実証している。具体的には、32 $\times$ 32グリッドで障害物密度20%の400エージェントのベンチマークにおいて、既存のサブオプティマルな手法の多くが30秒のタイムアウトで失敗する中、LaCAMは中央値1秒で全てのインスタンスを解決した。

Preliminaries

Multi-Agent Pathfinding (MAPF) は、グラフ $G = (V, E)$ 上の $n$ 個のエージェント $A = \{1, \dots, n\}$ に対し、各エージェント $i$ が開始地点 $s_i$ から目標地点 $g_i$ へ到達する一連の経路 $\pi_i = (\pi_i[0], \pi_i[1], \dots, \pi_i[T])$ を割り当てる問題である。各時刻 $t$ において、エージェントは隣接頂点または現在の頂点に留まることができ、$\pi_i[t+1] \in \text{neigh}(\pi_i[t]) \cup \{\pi_i[t]\}$ を満たす必要がある。解は、同一時刻に同じ頂点に位置する vertex conflict ($\pi_i[t] = \pi_j[t]$) および、エージェントが互いの位置を入れ替える swap conflict ($\pi_i[t] = \pi_j[t+1] \land \pi_i[t+1] = \pi_j[t]$) が存在しない conflict-free な経路集合である。本稿では、各エージェントが目標に到達して留まり始める最短時刻 $T_i$ の総和である sum-of-costs (SOC) $\sum_{i \in A} T_i$ を解の質を評価する指標として用いる。また、全エージェントの位置のタプルを configuration と定義し、隣接する configuration 間の遷移が conflict-free であることを接続の条件としている。

Algorithm

LaCAMは、高レベル探索と低レベル探索の2層構造を持つマルチエージェント経路計画（MAPF）のための探索ベースのアルゴリズムである。高レベル探索は、スタックを用いた深さ優先探索（DFS）スタイルで構成（configuration）の列を探索し、各ノードは制約木（constraint tree）を保持する。低レベル探索は幅優先探索（BFS）を用いて制約木を構築し、エージェント $i$ が現在の頂点 $v$ から隣接頂点 $u \in \text{neigh}(v) \cup \{v\}$ へ移動するという制約を順次生成する。低レベルの探索深度がエージェント数 $|A|$ に達するまで制約を割り当て、生成された制約を満たす新しい構成を `get new config` 関数（PIBTなどの既存MAPFアルゴリズムを応用可能）によって決定し、高レベルの `Open` リストへ追加する。本手法は、高レベルの探索空間が $O(|V|^{|A|})$、低レベルの構成生成が $O(\Delta^{|A|})$ であることから、解が存在するMAPFインスタンスに対して完全性（completeness）を持つ。実験では、PIBTを構成生成に用いることで、複雑な小規模インスタンスにおいて既存の $A^*$ や LNS2 よりも高速に解を導出できることが示されている。
