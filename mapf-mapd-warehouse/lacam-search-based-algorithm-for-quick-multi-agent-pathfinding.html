一言で: LaCAM (Lazy Constraints Addition Search) は、完全性を備えつつ、数百から数万規模のエージェントが存在する大規模なマルチエージェント経路探索（MAPF）問題を高速に解決する2レベル探索アルゴリズムである。高レベルの構成（configuration）探索と、低レベルの制約（constraint）探索を分離し、制約を遅延的に適用することで探索空間の爆発を抑制している。

どんなもの？

マルチエージェント経路探索（MAPF）は、グラフ $G = (V, E)$ 上の $n$ 個のエージェント $A = \{1, \dots, n\}$ が、各々の開始地点 $s_i$ から目標地点 $g_i$ へ衝突を回避しながら移動する経路 $\{\pi_1, \dots, \pi_n\}$ を求める問題である。回避すべき衝突には、同一時刻の頂点重複（vertex conflict: $\pi_i[t] = \pi_j[t]$）と、隣接時刻の入れ替わり（swap conflict: $\pi_i[t] = \pi_j[t+1] \land \pi_i[t+1] = \pi_j[t]$）がある。解の質は、各エージェントの目標到達時刻 $T_i$ の総和である sum-of-costs (SOC) $\sum_{i \in A} T_i$ で評価される。本研究は、大規模かつリアルタイム性が求められる環境において、既存のサブオプティマルな手法を凌駕する速度と成功率を実現することを目指している。

先行研究と比べてどこがすごい？

本論文の主な貢献は、完全性を維持しながら極めて高いスケーラビリティを実現するLaCAMアルゴリズムの提案である。LaCAMは、高レベル探索における分岐係数を実質的に抑制する「遅延的な制約適用」という新しいアプローチを導入している。実験により、400エージェントの高密度なベンチマークにおいて、既存手法がタイムアウトする中で中央値1秒という高速な解決能力を示すことを証明した。さらに、最大10,000エージェント規模の広大な環境においても、30秒以内に解を導出できる極めて高い拡張性を実証している。

技術や手法のキモはどこ？

LaCAMは、高レベルの構成探索と低レベルの制約探索からなる2層構造を採用している。高レベル探索はスタックを用いた深さ優先探索（DFS）スタイルで行われ、全エージェントの位置のタプルである「構成（configuration）」の列を探索する。低レベル探索は幅優先探索（BFS）を用いて制約木を構築し、エージェント $i$ が現在の位置 $v$ から隣接頂点 $u \in \text{neigh}(v) \cup \{v\}$ へ移動するという正の制約を順次追加していく。新しい構成の生成（`get_new_config`）にはPIBTなどの既存アルゴリズムを応用したブラックボックス関数を用い、制約を満たしつつ目標に有望な構成を生成する。高レベルの探索空間は $O(|V|^{|A|})$、低レベルの接続構成は $O(\Delta^{|A|})$ であり、解が存在する場合には必ず解を返す完全性が保証されている。

どうやって有効だと検証した？

評価実験では、PP、OD、PIBT、PIBT+、EECBS、LNS2といった既存のサブオプティマルなアルゴリズムを比較対象とした。MAPFベンチマークにおいて、特に障害物率20%の$32 \times 32$グリッド、エージェント数400の`random-32-32-20`シナリオにおいて、LaCAMは既存手法が失敗する中で成功率と実行時間の両面で優位性を示した。拡張性テストでは、`warehouse-20-40-10-2-2`において10,000エージェント規模のインスタンスをすべて30秒以内に解決した。一方で、幅1の通路でエージェントがすれ違う必要がある敵対的インスタンスでは、エージェント数の増加に伴いDFSが類似した構成に捕まり、探索回数が劇的に増大してタイムアウトするという限界も確認された。

議論はある？（限界・課題）

LaCAMの高速性の要因は、従来の $A^*$ が1つのノードから $O(\Delta|A|)$ 個の構成を生成して膨大な分岐を生むのに対し、高レベルノードから初期状態では1つの後続ノードのみを生成し、必要に応じて遅延的に他のノードを生成することで実質的な分岐係数を抑制している点にある。設計上の検討から、高レベル探索における再挿入（reinsert）操作がSOCの向上に寄与すること、およびPIBTによる後続状態生成が性能の鍵であることが明らかになった。今後の課題として、幅優先探索の採用によるmakespan最適解の取得や、大規模インスタンスに対する漸近的最適性（asymptotic optimality）を持つバージョンの開発、および低レベル探索における効果的な制約追加手法の改善が挙げられている。

Abstract

本論文では、マルチエージェント経路探索（MAPF）における新規かつ完全なアルゴリズムである LaCAM (Lazy Constraints Addition Search for MAPF) を提案している。LaCAMは、数百台以上のエージェントが存在する大規模な設定においても迅速に解を導出するため、2レベルの探索構造を採用している。低レベル探索ではエージェントの位置に関する制約を探索し、高レベル探索ではその制約に従いながら全エージェントの位置のシーケンスを探索する。網羅的な実験の結果、LaCAMは成功率、計画時間、および総コスト（sum-of-costs）の解の質において、既存の最先端な非最適（sub-optimal）MAPFアルゴリズムと同等またはそれ以上の性能を示すことが確認された。

Introduction

本論文では、大規模かつリアルタイム性が求められるマルチエージェント経路計画（MAPF）問題に対し、探索効率を劇的に向上させた新しいサブオプティマルなアルゴリズムであるLaCAM（Lazy Constraints Addition Search）を提案している。LaCAMは、全エージェントの配置のタプルである「構成（configuration）」を探索するハイレベルな探索と、次なる構成におけるエージェントの移動制約を探索するローレベルな探索からなる2層構造の探索フレームワークを採用しており、ハイレベルの構成生成時にローレベルの制約を遅延的に（lazy manner）適用することで探索コストを大幅に削減している。理論的には完全性（completeness）を備えており、解が存在する場合は必ず解を返し、存在しない場合はその旨を報告する。実験では、1491 $\times$ 656の広大なグリッドや10,000規模の大量のエージェント、高密度な環境を含む多様なインスタンスにおいて、極めて短時間での解決能力を示している。具体的には、障害物率20%の$32 \times 32$グリッドにおけるエージェント数400のベンチマーク問題において、既存のサブオプティマルな手法の多くが30秒のタイムアウトで失敗する中、LaCAMは中央値1秒という高速な実行時間で全インスタンスを解決した。

Preliminaries

Multi-Agent Pathfinding (MAPF) は、グラフ $G = (V, E)$ 上の $n$ 個のエージェント $A = \{1, \dots, n\}$ に対し、各エージェント $i$ が開始地点 $s_i$ から目標地点 $g_i$ へ移動する一連の経路 $\{\pi_1, \dots, \pi_n\}$ を求める問題である。各時刻 $t$ において、エージェント $i$ の位置 $\pi_i[t]$ は $\text{neigh}(\pi_i[t]) \cup \{\pi_i[t]\}$ のいずれかであり、同一時刻の重複（vertex conflict: $\pi_i[t] = \pi_j[t]$）および隣接時刻の入れ替わり（swap conflict: $\pi_i[t] = \pi_j[t+1] \land \pi_i[t+1] = \pi_j[t]$）を回避しなければならない。解の品質は、各エージェントが目標に到達して以降停止したとみなす最短時刻 $T_i$ の総和である sum-of-costs (SOC) $\sum_{i \in A} T_i$ によって評価される。すべてのエージェントの位置のタプルを configuration と呼び、2つの configuration $X, Y$ が接続しているとは、各エージェントの移動 $\{(X[1], Y[1]), \dots, (X[n], Y[n])\}$ が衝突を含まないことを指す。なお、グラフの最大次数を $\Delta$ と表記する。

Algorithm

LaCAMは、高レベル探索と低レベル探索の2層構造を持つマルチエージェント経路探索（MAPF）アルゴリズムである。高レベル探索は、スタックを用いた深さ優先探索（DFS）スタイルで構成（configuration）の列を探索し、各ノードはエージェントの次ステップの移動を規定する制約（constraint）を保持する制約木（constraint tree）を伴う。低レベル探索は幅優先探索（BFS）を用いて制約木を構築し、エージェント $i$ が現在の位置 $v$ から隣接頂点 $u \in \text{neigh}(v) \cup \{v\}$ へ移動するという制約を、エージェントの重複がないように順次追加していく。新しい構成の生成（`get_new_config`）はブラックボックス関数として扱われ、PIBTなどの既存のMAPFアルゴリズムを応用して、制約を満たしつつ目標に有望な構成を生成する。エージェントの優先順位は、初期状態ではスタートとゴールの距離の降順、探索中にはゴールに未到達のエージェントを優先するように設計されている。本手法は、高レベルの探索空間が $O(|V|^{|A|})$、低レベルの接続構成が $O(\Delta^{|A|})$ であることから、解が存在するMAPFインスタンスに対して完全性（completeness）を持つことが証明されている。

Evaluation

LaCAMの評価実験では、PP、OD、PIBT、PIBT+、EECBS、LNS2といった既存のサブオプティマルなMAPFアルゴリズムをベースラインとして、複雑な小規模インスタンス、MAPFベンチマーク、敵対的インスタンス、最大10,000エージェントの拡張性、および実装設計の5つの観点から検証が行われた。MAPFベンチマークにおいて、LaCAMはPPやEECBS、LNS2と比較して成功率と実行時間の両面で優位性を示し、特に400エージェントの`random-32-32-20`のような高密度なシナリオでベースラインが失敗する中、許容可能な解の質（SOC）を維持しつつ解を導出した。しかし、幅1の通路でエージェントがすれ違う必要がある敵対的インスタンスでは、エージェント数の増加に伴い探索回数が劇的に増大し、DFS形式の探索が類似した構成に捕まることでタイムアウトに至るという限界が示された。拡張性テストでは、`warehouse-20-40-10-2-2`において10,000エージェント規模でも全てのインスタンスを30秒以内に解決し、極めて高いスケーラビリティを実証した。また、設計の検討により、高レベル探索における再挿入（reinsert）操作がSOCの向上に寄与すること、およびPIBTによる後続状態生成がLaCAMの優れた性能の鍵であることを明らかにした。

Related Work

MAPF（Multi-Agent Pathfinding）のアルゴリズムは、エージェント間で結合して探索を行う「探索ベース」、SAT等の問題に帰着させる「コンパイルベース」、個別に順次計画する「優先順位付き計画」、およびルールに従い逐次移動する「ルールベース」の4種に分類される。LaCAMは探索ベースの手法に属するが、構成生成においてはPIBTのような優先順位付き計画やルールベースの手法も活用する。既存の主要なMAPFアルゴリズムは、高レベルで負の制約（どのエージェントがいつどこを使えないか）を探索し、低レベルで制約に従い単一エージェントの経路を探索する2レベル構造を持つが、LaCAMは制約を低レベルで扱い、実装では「誰がいつどこにいるべきか」を指定する正の制約を用いる。この正の制約の利用はCBS (Li et al. 2019) 等で見られるが、LaCAMにおいて負の制約を用いる場合と比較して、低レベルの探索空間を劇的に縮小できる利点がある。また、LaCAMは後続ノードの生成時に衝突判定のみを行う従来の $A^*$ 変種とは異なり、高レベルでの構成生成において他のMAPFアルゴリズムを組み込むことで積極的な協調（aggressive coordination）を実現しており、制約の追加手法は $A^*$ with operator decomposition (Standley 2010) の、全エージェントの行動ではなく単一エージェントの行動に対応する後続ノードを生成する仕組みに影響を受けている。

Conclusion and Discussion

本論文では、完全性と高速性を両立した新しい探索ベースのMAPFアルゴリズムであるLaCAMを提案している。LaCAMが高速である理由は、従来のA*が1つのノードから $O(\Delta|A|)$ 個の構成（configuration）を生成し分岐係数が膨大になるのに対し、高レベルノードから初期状態では1つの後続ノードのみを生成し、必要に応じて遅延評価（lazy manner）的に他の後続ノードを生成することで、実質的な分岐係数を抑制しているためである。実験では、他の最先端の劣最適ソルバーが対応できない複雑で高密度なシナリオにおいても、10,000エージェントのインスタンスを数十秒で解くという高いスケーラビリティを示した。今後の展望として、幅優先探索のスタイルを採用することでmakespan最適解を得る手法や、大規模なインスタンスに対して実用的な漸近的最適性（asymptotic optimality）を持つバージョンの開発が挙げられている。また、低レベル探索における効果的な制約の追加や、構成生成器（configuration generator）の設計改善が、現在の限界を克服するための重要な研究課題として示されている。
